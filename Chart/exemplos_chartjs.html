Chart.js — Exemplos Práticos

7 exemplos funcionais usando dados reais de chamados de suporte. Cada gráfico acompanha o código comentado linha a linha.

1

Barras Verticais — type: "bar"

O gráfico mais básico. Compara a quantidade de chamados por tipo (Bug vs Melhoria). Usa labels para o eixo X e data para os valores. Cada barra recebe uma cor diferente via backgroundColor como array.
ParâmetroValor usadoO que faz
type "bar" Define o tipo: barras verticais
labels ["Bug", "Melhoria"] Rótulos do eixo X. Cada posição = uma barra
data [7, 3] Valores numéricos. Posição 0 → primeiro label
backgroundColor ["#ef4444", "#6366f1"] Cor por barra. Array = uma cor para cada. String = cor única
borderRadius 8 Cantos arredondados (px)
borderSkipped false false = arredonda TODOS os cantos. "bottom" = só o topo
responsive true Redimensiona com a janela
maintainAspectRatio false false = preenche 100% do container (altura via CSS)
legend.display false Oculta a legenda (com 1 dataset, é redundante)
y.beginAtZero true Eixo Y começa em 0 (sem isso, pode começar em 3) 
// 1. Pega o <canvas> pelo id
var ctx = document.getElementById("chart1");

// 2. Cria o gráfico
new Chart(ctx, {
  type: "bar",

  data: {
    labels: ["Bug", "Melhoria"],
    datasets: [{
      label: "Quantidade de chamados",
      data: [7, 3],
      backgroundColor: ["#ef4444", "#6366f1"],
      borderRadius: 8,
      borderSkipped: false,
      barThickness: 80,
    }]
  },

  options: {
    responsive: true,
    maintainAspectRatio: false,
    plugins: { legend: { display: false } },
    scales: {
      x: { grid: { display: false } },
      y: { beginAtZero: true }
    }
  }
});

Barras Empilhadas — stacked: true + datalabels

Mostra Bug vs Melhoria empilhados por mês. A chave é scales.x.stacked: true e scales.y.stacked: true. Cada dataset vira uma camada. O plugin datalabels exibe os valores sobre cada segmento.
Bug Melhoria
ParâmetroValorO que faz
datasets 2 objetos Cada objeto = 1 camada de barras empilhadas
scales.x.stacked true Empilha barras no eixo X (sobrepostas, não lado a lado)
scales.y.stacked true Eixo Y soma todos os datasets para a altura total
datalabels.display function(ctx) {...} Função que oculta rótulos de valores 0
datalabels.anchor "center" Ancora o rótulo no centro da barra 
new Chart(ctx, {
  type: "bar",
  data: {
    labels: ["Jan/26", "Fev/26", "Mar/26", "Abr/26"],
    datasets: [
      { label: "Bug",      data: [1,2,2,2], backgroundColor: "#ef4444" },
      { label: "Melhoria", data: [1,0,1,1], backgroundColor: "#6366f1" }
    ]
  },
  options: {
    scales: {
      x: { stacked: true },  // empilha no X
      y: { stacked: true }   // empilha no Y
    },
    plugins: {
      datalabels: {
        display: function(ctx) {
          return ctx.dataset.data[ctx.dataIndex] > 0;
        },
        color: "#fff",
        anchor: "center",
        align: "center"
      }
    }
  }
});

Doughnut (Rosca) — type: "doughnut" + legenda customizada

Mostra a distribuição por categoria. cutout: "52%" controla o tamanho do buraco central. A legenda usa generateLabels para exibir a porcentagem ao lado de cada nome. O datalabels só aparece em fatias maiores que 8%.
ParâmetroValorO que faz
type "doughnut" Gráfico de rosca. Use "pie" para pizza sólida
cutout "52%" Tamanho do buraco central. "0%" = sólido, "80%" = fino
borderWidth 2 Borda entre as fatias (px)
hoverOffset 8 Quanto a fatia "salta" no hover (px)
legend.position "right" Posição da legenda: "top", "bottom", "left", "right"
generateLabels function Customiza texto da legenda: "Impostos (20.0%)" 
new Chart(ctx, {

  },
  options: {
    cutout: "52%",   // tamanho do buraco
    plugins: {
      legend: {
        position: "right",
        labels: {

            // Adiciona percentual à legenda
            return chart.data.labels.map(function(label, i) {

              return { text: label + " (" + pct + "%)", ... };

    }
  }

Gráfico de Linha — type: "line" + fill + tension

Mostra a evolução das horas gastas por mês. fill: true preenche a área abaixo da linha. tension: 0.35 suaviza as curvas. O backgroundColor usa opacidade HEX ("#6366f118" → ~9%) para preenchimento sutil.
ParâmetroValorO que faz
borderColor "#6366f1" Cor da linha
backgroundColor "#6366f118" Preenchimento. "18" = opacidade HEX (~9%)
fill true true = preenche abaixo da linha. false = só a linha
tension 0.35 Curvatura: 0 = reta, 0.35 = suave, 1 = muito curvo
pointRadius 6 Tamanho das bolinhas nos pontos
pointBackgroundColor "#fff" Branco = efeito de "bolinha vazada"
pointBorderWidth 2.5 Espessura da borda do ponto
y.ticks.callback v → v + "h" Formata os ticks do eixo: "4h", "8h", "12h" 
new Chart(ctx, {
  type: "line",
  data: {
    labels: ["Jan/26", "Fev/26", "Mar/26", "Abr/26"],
    datasets: [{
      label: "Horas gastas",
      data: [4.5, 7.0, 5.5, 11.0],
      borderColor: "#6366f1",         // cor da linha
      backgroundColor: "#6366f118",   // preenchimento ~9%
      fill: true,                      // preenche abaixo
      tension: 0.35,                   // curvatura suave
      pointRadius: 6,
      pointBackgroundColor: "#fff",   // bolinha vazada
      pointBorderWidth: 2.5,
      pointBorderColor: "#6366f1"
    }]
  },
  options: {
    scales: {
      y: {
        ticks: {
          callback: function(v) { return v + "h"; }
        }
      }
    }
  }
});

Barras Horizontais — indexAxis: "y"

Mostra o tempo médio por categoria. O segredo é indexAxis: "y" — uma única propriedade transforma barras verticais em horizontais. Quando ativo, scales.x vira o eixo de valores e scales.y vira o eixo de categorias.
ParâmetroValorO que faz
indexAxis "y" A ÚNICA mudança para barras horizontais. "x" = vertical (padrão)
scales.x Eixo de valores Com indexAxis: "y", X vira os VALORES (horizontal)
scales.y Eixo de categorias Com indexAxis: "y", Y vira as CATEGORIAS (vertical) 
new Chart(ctx, {
  type: "bar",
  data: {
    labels: ["Erro OCR", "Leitura CNPJ", "Valor monetário", ...],
    datasets: [{
      data: [5.5, 3.75, 2.5, 1.25, 1.25],
      backgroundColor: ["#6366f1", "#f59e0b", ...],
      borderRadius: 6
    }]
  },
  options: {
    indexAxis: "y",   // ← A MÁGICA: barras horizontais
    plugins: {
      datalabels: {
        formatter: function(v) { return v + "h"; }
      }
    }
  }
});

Múltiplas Linhas — 2 datasets tipo line

Duas linhas no mesmo gráfico: Bug (vermelho) e Melhoria (indigo). Cada dataset tem suas próprias cores. A função datalabels.color usa ctx.dataset.borderColor para que a cor do rótulo combine com a linha correspondente.
Bug Melhoria
ParâmetroValorO que faz
datasets 2 objetos Cada objeto = 1 linha com suas próprias cores e dados
datalabels.color function(ctx) Retorna ctx.dataset.borderColor → cor do rótulo = cor da linha
y.ticks.stepSize 1 Incremento do eixo Y (1 em 1, sem decimais) 
new Chart(ctx, {
  type: "line",
  data: {
    labels: ["Jan/26", "Fev/26", "Mar/26", "Abr/26"],
    datasets: [
      {
        label: "Bug",
        data: [1, 2, 2, 2],
        borderColor: "#ef4444",
        backgroundColor: "#ef444418",
        fill: true,
        tension: 0.35
      },
      {
        label: "Melhoria",
        data: [1, 0, 1, 1],
        borderColor: "#6366f1",
        backgroundColor: "#6366f118",
        fill: true,
        tension: 0.35
      }
    ]
  },
  options: {
    plugins: {
      datalabels: {
        // Cor do rótulo = cor da linha do dataset

Destruir e Recriar — chart.destroy() + referência global

O mesmo <canvas> é reutilizado para 3 tipos de gráfico. Antes de criar um novo, é obrigatório chamar chart.destroy() no anterior. Sem isso, o Chart.js sobrepõe gráficos e causa cintilação. A referência fica em uma variável global chartEx7.
MétodoO que faz
chart.destroy() Remove o gráfico do canvas e libera memória. OBRIGATÓRIO antes de criar novo
chart.update() Redesenha com dados atuais (se modificou chart.data.datasets diretamente)
chart.toBase64Image() Exporta o gráfico como imagem PNG em base64 
// Variável GLOBAL que guarda a referência ao gráfico
var chartEx7 = null;

function criarBarras() {
  var ctx = document.getElementById("chart7");

  // 1. DESTROI o gráfico anterior (se existir)
  if (chartEx7) {
    chartEx7.destroy();   // libera canvas + memória
    chartEx7 = null;
  }

  // 2. Cria o novo e GUARDA a referência
  chartEx7 = new Chart(ctx, {
    type: "bar",
    data: { ... },
    options: { ... }
  });
}

function criarRosca() {
  if (chartEx7) chartEx7.destroy();
  chartEx7 = new Chart(ctx, {
    type: "doughnut", ...
  });
}

function criarLinha() {
  if (chartEx7) chartEx7.destroy();
  chartEx7 = new Chart(ctx, {
    type: "line", ...
  });
}

Exportar JSON — Blob + createObjectURL

Todos os dados processados podem ser exportados como JSON. Usamos Blob para criar o arquivo na memória, URL.createObjectURL() para gerar um link temporário e simulamos um clique em um <a> invisível para disparar o download. Tudo 100% no navegador, sem servidor.
ConceitoO que faz
new Blob([string], {type}) Cria objeto binário na memória com o conteúdo do arquivo
URL.createObjectURL(blob) Gera uma URL temporária (blob:http://...) para o Blob
link.download = "nome.json" Define o nome do arquivo ao baixar
link.click() Simula clique no <a> para iniciar o download
URL.revokeObjectURL(url) Libera a memória da URL temporária após o download
JSON.stringify(obj, null, 2) Serializa com indentação de 2 espaços para legibilidade 
function exportarJSON() {
  // 1. Monta o objeto com os resultados
  var resultado = {
    meta: { exportadoEm: new Date().toISOString() },
    distribuicao: {
      porTipo: [{ nome: "Bug", quantidade: 7 }, ...],
      porCategoria: [...],
      porMes: [...]
    },
    dados: DADOS
  };

  // 2. Cria Blob (arquivo na memória)
  var blob = new Blob(
    [JSON.stringify(resultado, null, 2)],
    { type: "application/json" }
  );

  // 3. Cria URL temporária
  var url = URL.createObjectURL(blob);

  // 4. Cria <a> e simula clique
  var link = document.createElement("a");
  link.href = url;
  link.download = "analise_chamados.json";
  link.click();

  // 5. Libera memória
  URL.revokeObjectURL(url);
}
